# 摘要
提出了一种新的操作系统设计方案，每个模块按细粒度设计好，然后对于跨模块的调用使用一个门函数，数据共享也使用专门的方式，通过调用FlexOS的接口来实现。在编译的时候，根据用户的配置将每个库函数以不同的后端进行生成，保证隔离性。
# 背景
大部分操作系统都在设计的时候固定了安全与隔离策略，这就导致了一个TradeOff，微内核和多分离内核系统有更高的安全性，但牺牲了性能；而单地址空间的操作系统为了性能牺牲了安全性。而FlexOS旨在打破这样的固定策略，根据具体的应用生成灵活的隔离策略。
# 设计要点
+ 只服务于单目标的应用程序而不是通用应用程序
+ 所有的组件都是一个库，每个库都是平等的
+ 使用抽象的机制来处理具体的技术细节
+ 尽量不要牺牲太多性能

+ FlexOS组件隔离的粒度可以配置
+ 硬件隔离机制可配置
+ 软件强化和隔离机制可配置
+ 不要因为灵活性失去太多性能
+ 用户应该方便的了解FlexOS所允许的设计空间
# 系统流程
+ 用户提交一份配置方案给工具链，告诉工具链他想要的引用程序以及需要依赖的组件和库
+ 工具链选择一个后端
+ 工具链选择对应的库，并进行**重写**，将他们放在
+ 生成一个镜像
# 隔离实现
+ 基于高度模块化的Unikraft实现，跨库调用使用FlexOS的接口替换以保证安全性
+ 当两个模块在不同隔离域时，使用更安全的方法实现，在相同域时，直接分配到栈上+直接调用
